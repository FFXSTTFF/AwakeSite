# Инвентарь игрока + билдер отрядов — дизайн

Дата: 2026-07-14. Статус: утверждено пользователем (секции 1–4).
Задачи: #80 (билдер), #81 (инвентарь) — одна фича, два этапа, каждый этап — свой PR.

## Цель

Офицер собирает отряды по 5 человек драг-н-дропом из пула, видя по каждому бойцу
пять индикаторов и КД. Индикаторы не проставляются вручную — они выводятся из
инвентаря игрока и загруженных пруф-скринов сборок.

## Индикаторы (вычисляются, не хранятся)

| Флаг | Иконка | Условие |
|------|--------|---------|
| Био | знак биоопасности | в инвентаре есть броня «комбинированной» категории (Штурма, Альбатрос-перехватчик, Скиф, Зивкас MK2 и т.п.) |
| Боевая | щиток | в инвентаре есть броня боевой категории |
| Снайперка | прицел | в инвентаре есть оружие снайперской категории |
| Скорость | сапог | загружен пруф-скрин сборки на скорость |
| Живучесть | сердечко | загружен пруф-скрин сборки на живучесть |

Категории определяются по `ItemDto.Category` из базы предметов (синк stalzone,
`ItemCacheService`). Точные префиксы категорий («armor/combined» и т.д.) уточнить
при реализации; если категория не отсекает качество (мастерка/легенда) — добавить
фильтр по качеству (поле `Color`). КД — из `PlayerStatsSnapshot`.

## Этап 1: инвентарь и пруфы (профиль)

### Данные
- **PlayerInventoryItem**: Id, UserId, ItemId (string, ID предмета stalzone), CreatedAt.
  Категория/имя/иконка не дублируются — берутся из ItemCacheService по ItemId.
  Предметов у игрока сколько угодно (боевая + био одновременно = оба флага).
- **PlayerBuildProof**: Id, UserId, BuildType (Speed | Vitality), Image (bytea, ≤ 2 МБ),
  ContentType, UploadedAt. Уникальность (UserId, BuildType) — новая загрузка заменяет.
  Флаг встаёт сразу при загрузке (без очереди модерации); офицер может открыть скрин
  и удалить пруф — флаг снимается.

### API
- `GET /api/players/{id}/inventory` — предметы + вычисленные флаги. Member+;
  свой инвентарь доступен и Гостю.
- `POST /api/profile/inventory/items` `{ itemId }` / `DELETE .../items/{itemId}` — сам игрок.
- `POST /api/profile/build-proof` — multipart (type + image), валидация: тип файла
  (png/jpeg/webp), размер ≤ 2 МБ. Сам игрок.
- `GET /api/players/{id}/build-proof/{type}/image` — сам владелец и Officer+.
- `DELETE /api/players/{id}/build-proof/{type}` — сам владелец и Officer+
  (офицерское удаление = «пруф не принят»).

### UI (свой профиль)
Блок «Инвентарь и сборки»: строка флагов-бейджей (как увидит офицер), поиск
предметов через существующий ItemCombobox, добавленные предметы чипами
(иконка + имя + крестик), два слота загрузки скринов с превью.
Чужой профиль: флаги-бейджи для Member+; скрины — по правам выше.

## Этап 2: билдер отрядов

### Данные/правила
- Отряды — те же сущности Squad/SquadMember, что на странице «Отряды».
- Лимит состава: 5. Игрок ровно в одном отряде: уникальный индекс на
  SquadMember.UserId (миграция; дубликаты, если есть, чистятся миграцией —
  остаётся самое свежее членство).
- Перенос в другой отряд автоматически убирает из старого.

### API
- `GET /api/squads/builder` — Officer+: составы всех отрядов + пул (Member+ без
  отряда), по каждому бойцу: ник, аватар, флаги, КД.
- `POST /api/squads/{squadId}/move-member` `{ userId }` — Officer+: добавляет в отряд,
  предварительно убрав из текущего (если был); отклоняет при 5/5.
- Существующие Add/Remove/SetLeader остаются; страница «Отряды» для участников —
  просмотр, как сейчас.

### UI `/squads/builder` (Officer+)
- Слева пул: поиск по нику, сортировка по КД, компактные карточки с 5 индикаторами.
- Справа сетка отрядов: карточка отряда с 5 слотами, счётчик 4/5, полный отряд
  подсвечен.
- Драг-н-дроп: **@dnd-kit/core** (единственная новая зависимость; выбрана из-за
  поддержки тач-устройств). На мобиле дублирующая кнопка «→ в отряд» (меню выбора).
- Ховер/тап по бойцу — попап: расшифрованные индикаторы + КД.
- Кнопка входа в билдер — на странице «Отряды», видна Officer+.

## Права (сводно)

| Действие | Кто |
|----------|-----|
| Свой инвентарь/пруфы: смотреть и менять | сам игрок (любой ранг) |
| Чужие флаги-индикаторы | Member+ |
| Чужие пруф-скрины: смотреть/удалять | Officer+ (и сам владелец) |
| Билдер (смотреть и редактировать) | Officer+ |
| Страница «Отряды» (просмотр) | Member+ (как сейчас) |

## Ошибки и краевые случаи
- Предмет из инвентаря пропал из базы stalzone (пересинк) → предмет показывается
  как «неизвестный», флаги по нему не считаются; игрок может удалить.
- Загрузка не-картинки или > 2 МБ → 400 с понятным текстом.
- move-member в полный отряд → 400 «Отряд укомплектован (5/5)».
- Конкурентное редактирование двумя офицерами → последний выигрывает; билдер
  рефетчит состояние после каждой операции (и при ошибке).
- У бойца нет снапшота статистики → КД показывается как «—».

## Тестирование
- Unit: деривация всех 5 флагов (в т.ч. «неизвестный предмет»), валидация загрузки,
  права на пруфы, move-member (лимит 5, автоснятие из старого, отряд не найден).
- Ручная приёмка: профиль (добавление предметов, загрузка скринов), билдер
  (днд на десктопе, кнопка на мобиле, попап с КД).

## Вне скоупа
- Очередь модерации пруфов (флаг встаёт сразу).
- История изменений составов.
- Автоподбор состава под тип карты (возможное развитие).
- Бусты (#82) и OCR клановых войн (#83) — отдельные фичи.
