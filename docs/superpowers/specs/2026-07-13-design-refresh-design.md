# Дизайн: обновление визуального стиля сайта + публичный лендинг

Дата: 2026-07-13. Статус: утверждено пользователем (секции 1–4).

## Цель

Сайт клана Awake [LOVE] выглядит сухо и пусто, мобильная версия неудобна. Обновляем
визуальный язык по референсу https://lunar-zone.com/ (атмосферный тёмный стиль,
крупная типографика, «живые» карточки), сохраняя палитру Static Green. Добавляем
публичный лендинг — сейчас `/` сразу редиректит на логин, публичного «лица» у клана нет.

Не входит в объём: светлая тема, смена палитры, i18n-доработки, генерация арта
(место под него оставляем), редизайн бэкенда (кроме одного нового эндпоинта).

## Референс: что берём у Lunar

- Почти чёрный фон с едва заметной сеткой-паттерном и мягкими радиальными свечениями
  за ключевыми блоками.
- Hero: бейдж-пилюля над заголовком, очень жирный крупный заголовок с цветными
  акцентными словами, подзаголовок, две CTA-кнопки, справа — фирменный круг/декор.
- Карточки: заметное скругление, иконка в тонированной цветной плитке, стрелка ↗
  в углу, hover-подсветка рамки.
- Центрированные заголовки секций с приглушённым подзаголовком, щедрые отступы.

Всё — в зелёной гамме Awake, не в синей гамме Lunar.

## 1. Визуальный язык («Static Green, оживлённый»)

Палитра без изменений: фон `#0e0e0f`, карточки `#161618`, рамки `#1f1f22`,
текст `#f0ede8` / muted `#6b6b72`, акцент `#3ddc84` (HSL-токены уже в `index.css`).

Новое:
- **Фон**: CSS-сетка-паттерн по всему сайту (linear-gradient, шаг ~64px,
  непрозрачность ~3%) + радиальные зелёные свечения за hero и заголовками секций
  (radial-gradient, ~8–12% непрозрачности). Точечно и приглушённо, не неон.
  Прежнее правило «без свечений» отменено этим решением (референс пользователя).
- **Шрифт**: Inter (variable, с кириллицей), self-hosted через `@fontsource-variable/inter`
  (CSP/офлайн-дружелюбно). Заголовки: `font-extrabold`/`font-black`, `tracking-tight`,
  акцентные слова — зелёным.
- **Карточки**: `rounded-xl`, иконка в плитке `bg-accent/10 rounded-lg`, стрелка ↗
  в углу интерактивных карточек, hover: `border-accent/30` + подъём на 2px.
- **Кнопки**: primary — зелёная заливка, на hover мягкое свечение
  (`shadow-[0_0_20px_rgba(61,220,132,0.3)]`).
- **Анимации**: появление секций при скролле — fade-in снизу через
  IntersectionObserver + CSS-классы (без framer-motion и т.п.); переходы цветов/теней
  150–200 мс.
- **Загрузка**: скелетоны (shadcn Skeleton) вместо текста «Загрузка…».
- **Пустые состояния**: иконка + текст + CTA (например «Тикетов пока нет — создай первый»).

## 2. Публичный лендинг `/`

`/` перестаёт редиректить на `/login` и становится публичной страницей.
Авторизованный пользователь видит лендинг с кнопкой «В дашборд» вместо «Войти».

Структура:
1. **Навбар** (fixed, blur-подложка): лого-точка + «Awake [LOVE]», справа «Войти»
   (или «В дашборд»).
2. **Hero**: пилюля «• КЛАНОВАЯ ПЛАТФОРМА STALCRAFT», заголовок с зелёным акцентным
   словом, подзаголовок про клан, CTA «Войти через Discord» + «Как вступить»
   (скролл к секции). Справа CSS-декор: зелёное кольцо с тегом [LOVE] —
   зарезервированное место под будущий сгенерированный арт.
3. **Топ игроков**: топ-10 участников по последним снапшотам статистики
   (ник, киллы, часы, точность) из нового публичного эндпоинта. На мобиле — карточки,
   на десктопе — таблица в новом стиле.
4. **Как вступить**: 3 шага-карточки (заявка в Discord → собеседование →
   доступ к платформе) + кнопка Discord-сервера клана. Ссылка-приглашение — константа
   `VITE_DISCORD_INVITE_URL`; пока пользователь её не дал, кнопка не рендерится
   (секция работает без неё).
5. **Футер**: тег клана, Discord, копирайт.

Логин-страница получает тот же фон/атмосферу; карточка остаётся.

## 3. Внутренние страницы + мобильная версия (этап 2)

- **Sidebar** (десктоп ≥768px): активный пункт — зелёная вертикальная полоса +
  подсветка, плавные hover-переходы, аккуратный блок пользователя.
- **Мобильная навигация** (<768px): нижняя таб-панель — Дашборд, Отряды, Тикеты,
  Профиль, «Ещё». «Ещё» открывает лист с настройками, управлением (по рангу) и
  выходом. Гамбургер и мобильное боковое меню удаляются.
- **Контентные страницы** (дашборд, профиль, отряды, тикеты, управление):
  единые заголовки/отступы, карточки в новом стиле, скелетоны, пустые состояния;
  таблицы на мобиле сворачиваются в карточки (сейчас вылезают за экран).
- **Профиль игрока**: статистика как стат-плитки — крупная цифра, подпись, иконка.

## 4. Бэкенд: публичный топ игроков

`GET /api/public/leaderboard` (новый `PublicController`, `[AllowAnonymous]`):
- Запрос `GetLeaderboardQuery` (MediatR): по каждому пользователю ранга Member и выше
  с привязанным игровым ником берётся его последний снапшот статистики;
  сортировка по киллам, top-10.
- DTO — только публичные поля: игровой ник, киллы, часы, точность. Никаких
  Discord-идентификаторов, рангов доступа и внутренних id.
- Кэш в памяти 5 минут (`IMemoryCache`), чтобы лендинг не грузил БД.
- Unit-тесты на хэндлер (сортировка, ограничение количества, пустая БД).

## Этапность и ветка

Новая ветка от свежего master (PR #8 смержен).

- **Этап 1 — фундамент + лендинг** (утверждение стиля вживую): токены/фон/шрифт,
  рестайл базовых ui-компонентов (button, card, skeleton), лендинг, логин,
  эндпоинт leaderboard. Пользователь смотрит в dev-стеке → правки → одобрение стиля.
- **Этап 2 — внутренние страницы**: sidebar, нижняя панель, контентные страницы,
  скелетоны/пустые состояния/мобильные таблицы.

Этап 2 не начинается, пока стиль этапа 1 не одобрен.
